# DevX Artifacts Implementation Plan

Date: 2026-04-25
Status: draft, updated with product decisions
Branch/worktree: `jf-add-artifacts`

## Summary

Add a first-class artifact system to DevX so any agent or human can attach rich files to a session, view them in DevX Web next to the terminal, reference them from prompts, and archive durable outputs during session cleanup.

The implementation should be staged so the CLI + filesystem convention lands first, then the web read/view path, then upload/autocomplete, then archive browsing and polish.

## Current Repo Context

Relevant existing surfaces:

- CLI commands are Cobra commands in `cmd/`, with command families split by file (`cmd/session_*.go`, `cmd/web.go`, etc.).
- Session metadata is managed in `session/metadata.go` and includes session name, worktree path, routes, display details, and attention flags.
- Current session detection already exists via `session.GetCurrentSessionName()`.
- Session removal flow is in `cmd/session_rm.go` and is the right place to archive retained artifacts before worktree deletion.
- Attention flags already exist end-to-end:
  - CLI: `cmd/session_flag.go`
  - session metadata: `session.SetAttentionFlag`, `session.ClearAttentionFlag`
  - web SSE/API: `web/api.go`, `web/sse.go`
  - UI notifications: `web/app/src/App.svelte`, `web/app/src/lib/FlagToast.svelte`
- DevX Web already serves an authenticated Svelte SPA and authenticated API routes:
  - server/router: `web/server.go`, `web/api.go`
  - frontend root: `web/app/src/App.svelte`
  - terminal component: `web/app/src/lib/Terminal.svelte`
  - frontend API client: `web/app/src/api.js`
- Existing image upload/drop/paste behavior in `Terminal.svelte` and `/api/upload-image` is a useful pattern for human uploads, but artifacts should supersede/generalize it rather than remain image-only.
- `.gitignore` currently ignores `.devx/`, `.envrc`, `.tmuxp.yaml`, etc., but not `.artifacts/`.

## Proposed Package/Layout

Add a dedicated package for artifact domain logic so CLI and Web share behavior.

```text
artifact/
  manifest.go       # schema, load/save manifest, validation
  paths.go          # artifact dir/path resolution, safe joins
  add.go            # add/register/update artifact
  detect.go         # type/format detection, slug/id generation
  archive.go        # archive retained artifacts
  theme.go          # embedded default theme.css writer
  upload.go         # helpers for uploaded file/text artifacts
  *_test.go

cmd/
  artifact.go
  artifact_add.go
  artifact_list.go
  artifact_remove.go
  artifact_url.go
  artifact_archive.go

web/
  artifacts_api.go  # API/static handlers for session artifacts

web/app/src/lib/artifacts/
  ArtifactDropdown.svelte
  ArtifactPane.svelte
  ArtifactViewer.svelte
  ArtifactUploadModal.svelte
  ArtifactSearchOverlay.svelte
  artifactTypes.js
```

Rationale: artifact operations must be usable from CLI, session cleanup, and web upload endpoints without shelling out to `devx artifact add` internally.

## Data Model

### Manifest structs

```go
type Manifest struct {
    Version   int        `json:"version"`
    Session   string     `json:"session"`
    Artifacts []Artifact `json:"artifacts"`
}

type Artifact struct {
    ID        string    `json:"id"`
    Type      string    `json:"type"`
    Title     string    `json:"title"`
    File      string    `json:"file"`
    Created   time.Time `json:"created"`
    Agent     string    `json:"agent,omitempty"`
    Retention string    `json:"retention,omitempty"`
    Summary   *string   `json:"summary,omitempty"`
    Tags      []string  `json:"tags,omitempty"`
    Focus     bool      `json:"focus,omitempty"`
}
```

### Allowed values

Artifact types:

- `plan`
- `report`
- `screenshot`
- `recording`
- `log`
- `diff`
- `document`
- `other`

Retention:

- `session` default
- `archive`

### Path safety requirements

All artifact file paths must be relative to `.artifacts/` and must pass safe path validation:

- reject absolute paths in manifest file values
- reject `..` path components
- reject empty paths
- preserve subdirectories only when generated by known rules or explicitly allowed
- use `filepath.Clean`, then verify the final absolute path remains within the artifact dir

This is especially important for web static serving and upload endpoints.

## Phase 1: Foundation CLI + Directory Convention

Goal: agents can create, list, remove, URL, and mark artifacts from the terminal. Web does not need split pane yet.

### 1. Add `.artifacts/` to gitignore defaults

- Update repository `.gitignore` to include `.artifacts/`.
- If there is a project initialization path that writes default gitignore/devx templates, update it too after locating the relevant code.

### 2. Implement artifact package foundation

Add `artifact/` package with:

- `DirForSession(sess *session.Session) string` → `<sess.Path>/.artifacts`
- `LoadManifest(sess *session.Session) (*Manifest, error)`
  - missing manifest returns empty manifest `{ version: 1, session: sess.Name }`
- `SaveManifest(sess *session.Session, manifest *Manifest) error`
  - writes atomically: temp file then rename
- `ValidateArtifact(a Artifact) error`
- `ValidateType`, `ValidateRetention`
- `DetectType(path string) string`
- `GenerateID(type, title, now) string`
- `Slugify(title) string`
- `ArtifactByID(manifest, id)`
- `SortByCreatedDesc` helpers

Tests:

- missing manifest creation
- malformed manifest errors
- safe path rejection
- type detection
- ID generation stable formatting
- duplicate IDs rejected or treated as update depending on final semantics

### 3. Default `theme.css`

- Add embedded default CSS in `artifact/theme.go` or as an embedded file under `artifact/theme.css`.
- On first artifact creation, write `.artifacts/theme.css` if it does not exist.
- Do not overwrite a user-customized theme.

### 4. `devx artifact add`

Command:

```bash
devx artifact add [flags] <file-or->
```

Flags:

- `--type string`
- `--title string` required
- `--summary string`
- `--agent string` default `$DEVX_AGENT` or `unknown`
- `--retention string` default `$DEVX_ARTIFACT_RETENTION` or `session`
- `--tags string`
- `--focus`
- `--id string`
- `--file string` destination filename, needed for stdin support
- optionally `--session string` for cross-session add from outside the worktree
- optionally `--move` if we want move semantics later; default should be copy for safety

Behavior:

1. Resolve target session:
   - if `--session`, load it from metadata
   - else use `session.GetCurrentSessionName()`
   - if still empty, return a helpful error: run from a devx worktree or pass `--session`
2. Validate title, type, retention, tags.
3. Resolve source:
   - regular file path: copy contents
   - `-`: read stdin; require `--file`
4. Determine destination relative path:
   - if `--file`, use it after validation
   - else type-based subdirectory:
     - screenshots → `screenshots/<basename>`
     - recordings → `recordings/<basename>`
     - logs → `logs/<basename>`
     - other types → `<basename>`
   - avoid overwrite by appending `-2`, `-3`, etc. unless updating an existing ID intentionally
5. Ensure `.artifacts/` and needed subdirs exist.
6. Write/copy file.
7. Ensure `theme.css` exists.
8. Append manifest entry or update existing entry if `--id` matches an existing artifact.
9. If `--focus`:
   - set `Focus=true` on new artifact
   - set `Focus=false` on other artifacts in same manifest, so there is one current focus target
   - call `session.SetAttentionFlag(sessionName, "New artifact: "+title)` directly or via existing CLI helper
   - notify web server with existing flag notification mechanism if feasible
10. Print URL.

Open implementation detail: whether duplicate `--id` with a new file should replace the old file. Recommended: allow update only with explicit same `--id`; replace metadata and copy new file, but leave old file if the file path changes unless `devx artifact remove` is used.

Tests:

- add file creates `.artifacts/manifest.json`
- add stdin requires `--file`
- auto type detection by extension
- tags parsing trims spaces and drops empty tags
- focus sets attention flag and manifest focus
- duplicate title generates unique ID by timestamp or errors on exact duplicate ID
- path traversal through `--file` rejected

### 5. `devx artifact list`

Command:

```bash
devx artifact list [flags]
```

Flags:

- `--type string`
- `--tag string`
- `--agent string`
- `--search string`
- `--json`
- `--session string`

Behavior:

- resolve session similarly to add
- load manifest
- filter by type/tag/agent/search
- default output as table sorted newest first
- JSON output should include computed fields:
  - `path`: `.artifacts/<file>` for local agent use
  - `url`: relative web URL, e.g. `/sessions/<encoded>/artifacts/<file>`

Tests:

- empty list output
- filters
- JSON shape includes path and URL

### 6. `devx artifact remove`

Command:

```bash
devx artifact remove [--session name] <id>
```

Behavior:

- find artifact by ID
- delete file if it exists and is inside artifact dir
- remove manifest entry
- save manifest
- if removed artifact was focused, leave no focused artifact

Tests:

- removes manifest entry and file
- missing ID returns useful error
- unsafe manifest file path refuses deletion outside artifact dir

### 7. `devx artifact url`

Command:

```bash
devx artifact url [--session name] [--absolute] <id>
```

Behavior:

- find artifact
- default URL:
  - same-session relative file path for embedding could be either `./<file>` or web path. Need final decision.
  - Recommended split:
    - default prints local path `.artifacts/<file>` for prompt insertion?
    - `--web` prints `/sessions/{session}/artifacts/{file}`
    - `--absolute` prints `https://{external_domain}/sessions/...`
  - The spec says default output is `/sessions/...`; use that unless you prefer a separate `--local`.
- absolute URL should use `external_domain` if set, otherwise `http://localhost:{web_port}`.
- path components must be URL-escaped safely, especially session names containing `/`.

Tests:

- relative web URL
- absolute external domain
- errors on missing artifact

### 8. `devx artifact archive`

Command:

```bash
devx artifact archive [--session name] <id>
```

Behavior:

- set `retention=archive` on artifact
- save manifest

Tests:

- updates retention

### 9. Archive retained artifacts on `session rm`

Add `artifact.ArchiveSessionArtifacts(sess)` before worktree removal in `cmd/session_rm.go`.

Archive destination:

```text
<project root>/.devx/archive/YYYY-MM-DD-session-name/
```

Project root resolution:

1. `sess.ProjectPath` when present
2. parent of `.worktrees` derived from `sess.Path`
3. current working directory as fallback only if safe

Behavior:

- load manifest
- filter `retention == archive`
- copy files and local assets
- write archive manifest containing only archived entries
- do not fail session removal hard by default; print warning and continue, unless we decide archive failures should block deletion

Local asset copy strategy for v1:

- Always copy the artifact file.
- Also copy `theme.css` if present.
- For HTML/MD artifacts, conservative option: copy the entire `.artifacts/` directory subset? That can accidentally archive session-retained screenshots.
- Recommended v1 pragmatic strategy:
  - copy artifact file
  - copy `theme.css`
  - copy any sibling files/directories referenced by relative `src="..."` or `href="..."` in HTML, and markdown image/link references
  - if parsing feels too much for v1, document that archive only preserves the file and theme; add asset-copy in polish.

Tests:

- session artifact retained by archive is copied before worktree deletion
- session-retained artifact is not copied
- archive manifest includes only archived entries

## Phase 2: DevX Web Read/API/Viewer

Goal: artifacts are visible and render next to the terminal. Upload can wait.

### 1. Backend API routes

Add in `web/artifacts_api.go` and register from `registerAPIRoutes` / `Server.registerRoutes` as needed.

Endpoints:

```text
GET /api/artifacts?session={name}
GET /api/artifacts/item?session={name}&id={id}
GET /api/artifacts/file?session={name}&id={id}
```

Because this repo already avoids path params for session names containing `/`, prefer query parameters for API endpoints rather than `/api/artifacts/{session}` in the internal implementation.

If preserving spec URLs externally matters, static route can still be:

```text
/sessions/{encoded-session}/artifacts/{file}
```

But beware session names with slashes. Recommended static serving design:

```text
/sessions/{url.PathEscape(session-name)}/artifacts/{relative-file}
```

Implementation details:

- Authenticate `/sessions/` static artifacts too. Update `authMiddleware` to require auth for `/sessions/` artifact routes.
- Resolve session via metadata and validate name with existing session validation.
- Use artifact package safe join.
- Use `http.ServeFile` for files.
- Manifest API response includes computed `url` and maybe `path` fields.

Tests:

- API requires valid session
- manifest returned
- file serving rejects traversal
- static route auth covered at server middleware level if possible

### 5. Artifact viewer component

Create `ArtifactViewer.svelte`.

Render by type/file extension:

- HTML: sandboxed iframe
  - `sandbox="allow-same-origin"` maybe no scripts by default. If scripts in generated reports are desired, use `allow-scripts` only after deciding risk.
  - `src` points at static artifact URL.
- Markdown: fetch text then render client-side. For v1, either:
  - add a small markdown package to `web/app`, or
  - render escaped plaintext with minimal heading/link handling. Recommended: use `marked` + sanitize if adding dependency is acceptable.
- Images: `<img>` with object-fit contain and click-to-zoom state.
- Video: `<video controls>`.
- Logs/text/diff: fetch text and show monospace pre. Line numbers/search can be v2 inside Phase 2.
- PDF/document: iframe browser native viewer.
- Other: download/open link plus raw preview if text.

Security:

- do not inject artifact HTML into DOM directly
- iframe HTML artifacts
- text previews escape content by default

### 6. Auto-open focused artifact

When `session.attention_flag` is true and manifest has an artifact with `focus=true`, open artifact pane and show a toast.

## Phase 3: Upload + Prompt Referencing

Endpoint:

```text

```

Fields:

- repeated `file`
- or `text`
- `title`
- `type`
- `format`
- `tags`
- `retention`



## Key Risks and Decisions Needed

1. **URL path design for session names containing `/`**
   - Existing web code avoids path params for session names because branch-style names contain slashes.
   - For static artifact URLs, we need robust escaping or an alternate URL shape.
   - Decision: API uses query params where practical; static uses `encodeURIComponent(session.name)` and server decodes first path segment from `RawPath`, same as terminal proxy.

2. **Shortcut conflict**
   - `Ctrl+Shift+S` already focuses the session list in the terminal iframe.
   - Decision: keep the existing shortcut and use `Ctrl+Shift+O` for artifact orientation/split cycle, with `Ctrl+Shift+A` still toggling artifacts.

3. **Default `artifact url` output**
   - Decision:
     - `devx artifact url <id>` → web path `/sessions/.../artifacts/...`
     - `devx artifact url --local <id>` → `.artifacts/<file>`
     - `devx artifact url --embed <id>` → `./<file>` for same-session HTML embedding
     - `--absolute` → full external URL

4. **Archive asset copying depth**
   - Decision: archive must preserve referenced local images/assets, not only the primary file.
   - Implement referenced-asset discovery for HTML and Markdown artifacts in the archive-on-remove slice.
   - Always copy `theme.css` when present.

5. **HTML sandbox policy**
   - Decision: JavaScript is allowed for HTML artifacts because rich reports may need interactivity.
   - Start with sandboxed iframes using `allow-scripts` but not `allow-same-origin`; revisit only if relative asset/rendering needs require it.

6. **Session removal archive failures**
   - Decision: archive failures should print a warning but should not block `devx session rm`.

7. **Current image upload behavior**
   - Existing `[img]` upload injects absolute paths from `~/.devx/uploads` and is useful for agents.
   - Artifacts are related but not identical.
   - Recommendation: do not remove `[img]` in v1; add artifact flows alongside it, then consider unifying later.

7. **Manifest update semantics**
   - Should `artifact add --id existing` update in place, error, or require `--replace`?
   - Recommendation: update metadata and replace file only when `--id` is explicitly provided; auto-generated ID collisions should error.

## Product Decisions from Jon

1. HTML artifact JavaScript is allowed.
2. `devx artifact url <id>` should print the web path by default.
3. Do not reuse `Ctrl+Shift+S`; use `Ctrl+Shift+O` for split cycling.
4. Archive failures during `devx session rm` should warn and continue.
5. Archived artifacts should preserve referenced images/assets.
6. Prioritize the full web upload/reference flow; the web side is the most useful side.
7. Do not include `allow-same-origin` in artifact HTML iframes for now; revisit if needed.
8. Batch upload with shared metadata is acceptable for v1, as long as artifacts can be renamed/edited in the web UI.
